# Product Strategy

The product surface should stay narrow: artifact-backed ranking, evidence lookup, and governed reporting.

## Core Product Boundary

- score backbones
- explain scores
- expose supporting evidence
- publish contract-backed release artifacts

## Things To Avoid

- unverifiable live claims
- write-heavy workflow surfaces in the public API
- product messaging that outruns the scientific contract

## Operating Rule

Ship improvements that reduce ambiguity, verification cost, or release risk before shipping new breadth.
